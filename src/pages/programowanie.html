GIT

Nauczyłem się podstawowych koment i pracy z repozytorium.

    Podstawowe:
  • Komenda do nadania imienia i nazwiska:
    git config --global user.name “Imie Nazwisko”
  • nadanie e maila:
    git config --global user.email “email@email.com”

    git config --get user.name

    git config --get user.email

  • git init (dodaje nowy plik git do śledzenia)
  • git status (sprawdza aktualny status plików ;P

  • gitk (status wszystkich komentarzy)
Flow działania: git add . / git commit -m “opis zmian” / git push

Dzień 8 Box model

Część I

Dziś zgłębiłam tajniki “Snake mode on” pdk ;) display : block; - element wyświetla się na całą szerokość rodzica.
przykład_______________________________________________________
display : inline; - element zajmuję tylko potrzebne miejsce np.“tutaj”. W całym tekście.
Wysokości i szerokości ( width and height) działają przy użyciu display : block i display : inline-block.
Prostą wersją responsywności strony jest połaczenie 2 wartości. width : 100px; - ustawia na “twardo” wartość szerokości.
width : 100%; + max-width : 700 px - ustala szerokość na 700 px ale po zmniejszeniu szerokości “zawija” zawartość (taka podstawa responsywności).
Wartości : vw - szerokość w procentach ale bez dziedziczenia względem rodzica . dostosowuje się do wymiarów strony automatycznie. np width : 50vw;

Część II


div - nie ma znaczenia semantycznego. Używany jest w momencie braku odpowiedniego znacznika.
box-sizing : content-box - powierzchnia na zawartość się nie zmniejsza, każde dodanie zwiększa wielkość całego elementu (pudełka).
border-box - zmienia się powierzchnia na zawartość, ale zostają wymiary wcześniej określone.
position: relative - Tak pozycjonowany element staje się kontenerem dla wszystkich innych, które są absolutnie w nim pozycjonowane.
